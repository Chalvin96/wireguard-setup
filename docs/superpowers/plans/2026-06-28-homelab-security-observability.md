# Homelab Security & Observability Stack — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a fully automated Ansible project that deploys WireGuard+HAProxy on the VPS, Caddy+fail2ban+Promtail on the mini PC edge node, and a Loki+Prometheus+Grafana observability stack on a dedicated monitoring VM — replacing all existing shell scripts.

**Architecture:** All configuration is managed from the operator's laptop via Ansible roles targeting three host groups (vps, edge, monitoring). A `bootstrap.yml` playbook creates the `deploy` user on each host once. The top-level `site.yml` runs roles in dependency order. Secrets live in Ansible Vault encrypted in-repo.

**Tech Stack:** Ansible 2.15+, ansible-lint, WireGuard, HAProxy, Caddy (official Debian repo), fail2ban, Promtail (Grafana apt repo), Docker CE + Compose v2, Loki 3.1.0, Prometheus v2.53.0, Grafana 11.1.0, Node Exporter (apt), nftables

## Global Constraints

- All target hosts: Debian/Ubuntu with apt and systemd
- Ansible runs from operator laptop; all hosts reachable via LAN or public IP
- Ansible user: `deploy` with `NOPASSWD:ALL` sudo, SSH key auth only (`PasswordAuthentication no`)
- All secrets in `ansible/group_vars/all/vault.yml` (Ansible Vault AES-256)
- User config in `ansible/group_vars/all/config.yml` (gitignored; `.example` committed)
- Docker image tags pinned — no `latest`
- nftables manages packet filtering on VPS; ufw disabled by `vps-blocklist` role
- VPS uses a separate `inet blocklist` table (not `inet filter`) to avoid conflicts
- Caddy from official Cloudsmith apt repo, not Ubuntu default
- fail2ban thresholds: 3 hits/60s scanner paths, 20 hits/60s 404 flood, incremental banning
- Loki and Prometheus retention: 14 days
- Grafana datasources, dashboards, alerts provisioned via YAML files under `/opt/monitoring/grafana/provisioning/`
- WireGuard keypairs and `vps_ban` SSH key pre-generated by operator, stored in vault

### Task 2: bootstrap.yml — create deploy user on all hosts

**Files:**
- Create: `ansible/bootstrap.yml`

**Interfaces:**
- Produces: `deploy` user with passwordless sudo and SSH key on all hosts; required before `site.yml` can run

- [ ] **Step 1: Write `ansible/bootstrap.yml`**

```yaml
---
# Run once per host as your existing named user.
# ansible-playbook ansible/bootstrap.yml -u <your_user> --ask-pass --ask-become-pass
- name: Bootstrap deploy user
  hosts: all
  gather_facts: false
  become: true

  tasks:
    - name: Create deploy group
      ansible.builtin.group:
        name: deploy
        state: present

    - name: Create deploy user
      ansible.builtin.user:
        name: deploy
        group: deploy
        shell: /bin/bash
        create_home: true
        state: present

    - name: Create .ssh directory
      ansible.builtin.file:
        path: /home/deploy/.ssh
        state: directory
        mode: "0700"
        owner: deploy
        group: deploy

    - name: Add operator SSH public key
      ansible.posix.authorized_key:
        user: deploy
        key: "{{ lookup('file', '~/.ssh/id_ed25519.pub') }}"
        state: present

    - name: Configure passwordless sudo
      ansible.builtin.copy:
        dest: /etc/sudoers.d/deploy
        content: "deploy ALL=(ALL) NOPASSWD:ALL\n"
        mode: "0440"
        validate: visudo -cf %s

    - name: Disable password authentication
      ansible.builtin.lineinfile:
        path: /etc/ssh/sshd_config
        regexp: "^#?PasswordAuthentication"
        line: "PasswordAuthentication no"
        state: present
      notify: Restart sshd

  handlers:
    - name: Restart sshd
      ansible.builtin.service:
        name: ssh
        state: restarted
```

**Interfaces:**

- Produces: `inet blocklist` nftables table on VPS; `banagent` user; `/usr/local/bin/ban-ip`; `authorized_keys` with forced command

- [ ] **Step 1: Write `ansible/roles/vps-blocklist/files/ban-ip`**

```bash
#!/bin/bash
set -e
IP="$1"
TIMEOUT="$2"
[[ "$IP" =~ ^[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}$ ]] \
  || { echo "Error: invalid IP '$IP'"; exit 1; }
[[ "$TIMEOUT" =~ ^[0-9]+$ ]] \
  || { echo "Error: invalid timeout '$TIMEOUT'"; exit 1; }
nft add element inet blocklist banned "{ $IP timeout ${TIMEOUT}s }"
```

- [ ] **Step 2: Write `ansible/roles/vps-blocklist/templates/blocklist.nft.j2`**

```
#!/usr/sbin/nft -f
# Managed by Ansible — do not edit manually

flush table inet blocklist

table inet blocklist {
    set banned {
        type ipv4_addr
        flags timeout
    }

    chain input {
        type filter hook input priority -10; policy accept;
        ip saddr @banned drop
    }
}
```

- [ ] **Step 3: Write `ansible/roles/vps-blocklist/tasks/main.yml`**

```yaml
---
# Add a WireGuard peer to the VPS — run standalone, not part of site.yml.
# ansible-playbook ansible/add-client.yml --vault-password-file .vault_password
- name: Add WireGuard client
  hosts: vps
  become: true
  vars_prompt:
    - name: client_name
      prompt: "Client name (letters, numbers, hyphen, underscore only)"
      private: false

  tasks:
    - name: Validate client name
      ansible.builtin.assert:
        that: client_name | regex_search('^[a-zA-Z0-9_-]+$')
        fail_msg: "Invalid client name — use only letters, numbers, hyphen, underscore"

    - name: Check client does not already exist
      ansible.builtin.command:
        cmd: grep -c "# Client: {{ client_name }}" /etc/wireguard/wg0.conf
      register: existing
      changed_when: false
      failed_when: existing.rc not in [0, 1]

    - name: Abort if client already exists
      ansible.builtin.fail:
        msg: "Client '{{ client_name }}' already exists in wg0.conf"
      when: existing.stdout | int > 0

    - name: Generate WireGuard private key
      ansible.builtin.command: wg genkey
      register: client_privkey
      changed_when: false

    - name: Derive public key
      ansible.builtin.shell:
        cmd: "echo {{ client_privkey.stdout }} | wg pubkey"
      register: client_pubkey
      changed_when: false

    - name: Read current AllowedIPs
      ansible.builtin.command:
        cmd: grep -oP '10\.8\.0\.\K\d+' /etc/wireguard/wg0.conf
      register: used_octets
      changed_when: false
      failed_when: false

    - name: Compute next free IP octet
      ansible.builtin.set_fact:
        next_octet: >-
          {{ (range(2, 254) | list
              | difference(used_octets.stdout_lines | map('int') | list))
             | first }}

    - name: Add peer block to wg0.conf
      ansible.builtin.blockinfile:
        path: /etc/wireguard/wg0.conf
        block: |
          [Peer]
          # Client: {{ client_name }}
          PublicKey = {{ client_pubkey.stdout }}
          AllowedIPs = 10.8.0.{{ next_octet }}/32
        marker: "# {mark} ANSIBLE MANAGED PEER {{ client_name }}"

    - name: Apply peer without tunnel downtime
      ansible.builtin.shell:
        cmd: wg syncconf wg0 <(wg-quick strip wg0)
        executable: /bin/bash
      changed_when: true

    - name: Print client config
      ansible.builtin.debug:
        msg: |
          ═══════════════════════════════════════════════
          WireGuard config for: {{ client_name }}
          Save as {{ client_name }}.conf or scan as QR code
          ═══════════════════════════════════════════════

          [Interface]
          PrivateKey = {{ client_privkey.stdout }}
          Address = 10.8.0.{{ next_octet }}/24
          DNS = 1.1.1.1

          [Peer]
          PublicKey = {{ vault_wireguard_server_public_key }}
          Endpoint = {{ vps_public_ip }}:{{ wireguard_port }}
          AllowedIPs = 0.0.0.0/0
          PersistentKeepalive = 25
```
